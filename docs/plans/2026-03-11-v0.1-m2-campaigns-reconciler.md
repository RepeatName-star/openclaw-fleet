# Bulk Management v0.1 - Milestone 2 (Campaign API + Reconciler) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add Campaign CRUD APIs and a restart-safe reconciler that fans out instance-targeted tasks to every instance matching a selector, with per-instance per-generation idempotency and dynamic membership (best-effort).

**Architecture:** Campaigns are stored in `campaigns`. Per-instance per-generation state is stored in `campaign_instances` and enforces idempotency via primary key `(campaign_id, generation, instance_id)`. A reconciler loads open campaigns, evaluates the selector against current instance labels, creates missing `campaign_instances` rows, and schedules tasks by inserting into existing `tasks` (targeted to `instance_id`). Gate checks are stubbed as "always pass" in Milestone 2; Milestone 3 will replace it with facts-based blocking + probes.

**Tech Stack:** Fastify, TypeScript, Zod, Postgres (pg), Vitest + pg-mem.

**Prereqs:** Milestone 0 (schema) and Milestone 1 (labels + selector parser) completed.

Minimal shape (use Zod):

```ts
import { z } from "zod";
import type { FastifyInstance } from "fastify";
import type { Pool } from "pg";

const CreateSchema = z.object({
  name: z.string().min(1),
  selector: z.string().min(1),
  action: z.string().min(1),
  payload: z.record(z.unknown()).optional(),
  gate: z.record(z.unknown()).optional(),
  rollout: z.record(z.unknown()).optional(),
  expires_at: z.string().datetime().optional(),
});

export async function registerCampaignRoutes(
  app: FastifyInstance,
  opts: { pool?: Pool },
) {
  app.get("/v1/campaigns", async (_req, reply) => {
    if (!opts.pool) return reply.code(500).send({ error: "server not configured" });
    const res = await opts.pool.query(
      "select id, name, selector, action, generation, status, created_at, updated_at, closed_at, expires_at from campaigns order by created_at desc",
    );
    reply.send({ items: res.rows });
  });

  app.post("/v1/campaigns", async (req, reply) => {
    if (!opts.pool) return reply.code(500).send({ error: "server not configured" });
    const parsed = CreateSchema.safeParse(req.body ?? {});
    if (!parsed.success) return reply.code(400).send({ error: "invalid payload" });
    const row = await opts.pool.query(
      "insert into campaigns (name, selector, action, payload, gate, rollout, expires_at) values ($1,$2,$3,$4,$5,$6,$7) returning *",
      [
        parsed.data.name,
        parsed.data.selector,
        parsed.data.action,
        parsed.data.payload ?? {},
        parsed.data.gate ?? {},
        parsed.data.rollout ?? {},
        parsed.data.expires_at ?? null,
      ],
    );
    reply.send(row.rows[0]);
  });

  app.get("/v1/campaigns/:id", async (req, reply) => {
    if (!opts.pool) return reply.code(500).send({ error: "server not configured" });
    const { id } = req.params as { id: string };
    const res = await opts.pool.query("select * from campaigns where id = $1", [id]);
    if (!res.rowCount) return reply.code(404).send({ error: "not found" });
    reply.send(res.rows[0]);
  });

  app.post("/v1/campaigns/:id/close", async (req, reply) => {
    if (!opts.pool) return reply.code(500).send({ error: "server not configured" });
    const { id } = req.params as { id: string };
    const res = await opts.pool.query(
      "update campaigns set status = 'closed', closed_at = now(), updated_at = now() where id = $1 and status = 'open' returning id, status, closed_at",
      [id],
    );
    if (!res.rowCount) return reply.code(404).send({ error: "not found" });
    reply.send(res.rows[0]);
  });
}
```

**Step 3: Implement minimal reconciler**

Create `src/campaigns/reconciler.ts`:
- Import `parseLabelSelector` + `matchLabelSelector` from `src/labels/label-selector.ts` (Milestone 1).
- Query `campaigns` with `status='open'` and `expires_at is null or expires_at > now()`.
- Build `instancesWithLabels` as `{ instanceId, instanceName, labels: Record<string,string> }[]`.
- For each campaign:
  - Parse selector; if parse fails, skip scheduling (Milestone 4 will emit a `campaign.blocked` event).
  - For each matching instance:
    - `insert into campaign_instances (...) values (...) on conflict do nothing`
    - If new row, set `state='pending'` initially.
  - Mark removals:
    - For rows in `campaign_instances` for this `(campaign_id,generation)` where state in `('pending','blocked','queued')` and instance no longer matches, set `state='removed'` and `updated_at=now()`.
  - Schedule tasks:
    - Select campaign_instances for this campaign generation where `state='pending'` and `task_id is null`.
    - For each, insert into `tasks (target_type,target_id,action,payload)` returning `id`.
    - Update campaign_instances row with `task_id` and `state='queued'`.

Keep it simple and deterministic. Wrap per-campaign scheduling in a transaction if needed.

Export a single entrypoint:

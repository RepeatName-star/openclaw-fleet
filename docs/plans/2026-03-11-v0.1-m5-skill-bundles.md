# Bulk Management v0.1 - Milestone 5 (Remote Skill Bundle Distribution) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Distribute skill folders to instances without relying on OpenClaw Gateway internals: control plane stores `tar.gz` bundles in Postgres and sidecars can download + install them into `~/.openclaw-fleet/skills`, then patch OpenClaw config to load that directory as an extra skill dir.

**Architecture:**
- Control plane:
  - Stores bundles in `skill_bundles` (`tar.gz`, no size limit in v0.1).
  - Exposes endpoints to upload/list/get metadata and download content.
- Sidecar:
  - Implements `fleet.skill_bundle.install` action.
  - Downloads bundle via control plane using device token auth.
  - Verifies sha256, extracts into `~/.openclaw-fleet/skills/<bundleName>` using staging + rename (best-effort atomic swap).
  - Ensures OpenClaw loads extra dir via:
    - `config.get` to obtain `baseHash`
    - `config.patch` to append `~/.openclaw-fleet/skills` into `skills.load.extraDirs`
- Gate correctness:
  - Any successful bundle install invalidates `skills_snapshot` (Milestone 3 already marks invalidation on skill-changing actions).
  - The system must wait for `skills.status` probe writeback before allowing skill-dependent execution.

**Tech Stack:** Fastify, TypeScript, Postgres (pg), Node fs, Vitest.

**Prereqs:** Milestone 3 gate/facts/probe in place (skills invalidation + skills.status probe scheduling).

```ts
export async function installSkillBundle(params: {
  download: () => Promise<ArrayBuffer>;
  expectedSha256: string;
  bundleName: string;
  skillsRootDir: string; // ~/.openclaw-fleet/skills
  configGet: () => Promise<{ baseHash?: string }>;
  configPatch: (raw: string, baseHash?: string) => Promise<void>;
}): Promise<void>
```

Implementation (v0.1):
- `mkdir -p ~/.openclaw-fleet/skills`
- Download bytes
- Compute sha256, compare with expected (throw on mismatch)
- Extract tar.gz into staging dir:
  - `~/.openclaw-fleet/skills/.staging/<uuid>/...`
  - Use a JS tar library (preferred) or a minimal extractor implementation if already available.
- Best-effort atomic swap:
  - Rename staging dir to `~/.openclaw-fleet/skills/<bundleName>`
- Patch OpenClaw config to include extraDirs:
  - `const cfg = await configGet();`
  - build `raw` merge patch ensuring `skills.load.extraDirs` contains `~/.openclaw-fleet/skills`
  - call `configPatch(raw, cfg.baseHash)`

  - Return nothing.

To pass dependencies cleanly, extend `createExecutor` options with:

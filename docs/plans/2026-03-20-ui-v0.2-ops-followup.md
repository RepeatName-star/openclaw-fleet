# UI v0.2 Ops Follow-up Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Refine the operator console after real deployment feedback by fixing the remaining UX gaps in instances, tasks/audit, skill installation selectors, file editing, and campaign payload editing without changing the underlying v0.2 operator information architecture.

**Architecture:** Keep the current operator-facing navigation and APIs, but tighten the interaction model around them. Treat this as a follow-up polish pass with targeted backend support only where the current UI lacks the primitives to distinguish manual/system tasks or to list all instances for selectors. Prefer reusable modal, textarea, and selector behaviors so the fixes land once and improve multiple pages together.

**Tech Stack:** TypeScript, Fastify, React 18, Vite, Vitest, Testing Library, Postgres

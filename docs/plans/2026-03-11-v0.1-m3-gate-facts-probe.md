# Bulk Management v0.1 - Milestone 3 (Gate / Facts / Probe) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make Campaign scheduling deterministic and explainable by gating on 4 facts (`online`, `gateway_reachable`, `version`, `skills_snapshot`) and by driving probes with exponential backoff + Full Jitter (cap 5 minutes). Execution is frozen when gateway is unreachable, but probes are allowed to continue under backoff.

**Architecture:**
- Facts live on `instances` (DB) plus `online` in Redis heartbeat keys.
- Gate is pure "check + block" (no repair). When blocked due to stale/missing facts, the control plane schedules probes.
- Probes are tasks (instance-targeted) executed by Sidecar:
  - `fleet.gateway.probe` returns `{ gateway_reachable, openclaw_version }` using the connect `hello-ok.server.version`.
  - `skills.status` refreshes skills snapshot (already supported).
- Backoff state is stored in Postgres `instance_probe_states` so control-plane restarts do not cause probe storms.

**Tech Stack:** Fastify, TypeScript, Postgres (pg), Redis (online heartbeat), Vitest + pg-mem.

**Prereqs:** Milestone 2 reconciler exists and is scheduling tasks; Milestone 1 selector parser exists.

  probe_kind text NOT NULL,

**Step 3: Implement evaluateGate**

Create `src/gate/evaluate.ts`:

```ts
export type GateInputs = {
  online: boolean;
  gateway_reachable: boolean | null;
  gateway_reachable_at: Date | null;
  openclaw_version: string | null;
  openclaw_version_at: Date | null;
  skills_snapshot_at: Date | null;
  skills_snapshot_invalidated_at: Date | null;
};

export type GateConfig = {
  minVersion: string;
};

export function evaluateGate(inputs: GateInputs, config: GateConfig, now = new Date()) {
  // TTLs locked by docs/bulk-management.md
  // online: derived from heartbeat presence; no timestamp
  // gateway_reachable: 30s
  // version: 24h
  // skills_snapshot: 10m + invalidation
  // Return: { ok, blocked_reason, needs_probes: Array<"gateway"|"skills"> }
}
```

Rules:
- `online` must be true.
- `gateway_reachable` must be true and `gateway_reachable_at` within 30s.
  - If missing/stale/false: block and request probe kind `"gateway"`.
- `openclaw_version_at` must be within 24h and `openclaw_version` parseable and `>= minVersion`.
  - If missing/stale/unparseable/too-low: block and request probe kind `"gateway"` (same probe refreshes both).
- `skills_snapshot` must be fresh:
  - `skills_snapshot_at` within 10m
  - and not invalidated (`skills_snapshot_invalidated_at == null` OR `skills_snapshot_invalidated_at <= skills_snapshot_at`)
  - If stale/invalidated: block and request probe kind `"skills"`.

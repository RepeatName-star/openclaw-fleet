# Bulk Management UI (v0.1) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** Update the existing web UI to support all shipped v0.1 bulk-management features (labels, groups/selectors, campaigns, events/artifacts/export, skill bundles), in addition to the existing instances/tasks/skills/memory pages.

**Architecture:** Keep the current hash-based router and sidebar navigation. Add new pages for each v0.1 resource and extend the UI API client. Use simple table + modal forms consistent with the existing UI styling; no auth changes (single-tenant MVP).

**Tech Stack:** React 18 + Vite, TypeScript, Testing Library / Vitest.

---

## Global Success Criteria

- UI exposes CRUD flows for v0.1 objects:
  - Labels (per instance): list/upsert/delete business labels
  - Groups: list/create/update/delete + matches
  - Campaigns: list/create/update/close + jump to filtered events
  - Events: list + filters + export (JSONL/CSV) + artifact viewer
  - Skill Bundles: list/upload/download + quick install helpers
- UI-only: no new control-plane semantics; use existing endpoints from `docs/api.md`.
- Tests:
  - `pnpm test` passes
  - `pnpm ui:build` passes

# Bulk Management v0.1 - Milestone 6 (CLI) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Provide a CLI that uses the same control-plane APIs as the UI (no backdoor). This enforces UI/control separation and makes bulk operations scriptable.

**Architecture:** A small Node.js CLI (TypeScript) that calls HTTP endpoints under `/v1/*` using `fetch`. Keep dependencies at zero (no commander) for v0.1. Commands map 1:1 to APIs: labels, groups(selectors), campaigns, events export, artifacts fetch, skill bundles upload/download.

**Tech Stack:** TypeScript (Node ESM), built-in `fetch`, Vitest (light tests for argument parsing + client).

**Prereqs:** Milestones 1-5 APIs exist.

**Step 3: Implement client**

Create `src/cli/client.ts`:

```ts
type FetchLike = (input: string, init?: RequestInit) => Promise<Response>;

export function createFleetClient(opts: { baseUrl: string; fetch?: FetchLike }) {
  const fetcher = opts.fetch ?? fetch;
  const baseUrl = opts.baseUrl.replace(/\\/$/, "");

  async function getJson(path: string) {
    const res = await fetcher(`${baseUrl}${path}`);
    if (!res.ok) throw new Error(`request failed: ${path}`);
    return res.json();
  }

  async function postJson(path: string, body: unknown) {
    const res = await fetcher(`${baseUrl}${path}`, {
      method: "POST",
      headers: { "content-type": "application/json" },
      body: JSON.stringify(body ?? {}),
    });
    if (!res.ok) throw new Error(`request failed: ${path}`);
    return res.json().catch(() => ({}));
  }

  async function patchJson(path: string, body: unknown) {
    const res = await fetcher(`${baseUrl}${path}`, {
      method: "PATCH",
      headers: { "content-type": "application/json" },
      body: JSON.stringify(body ?? {}),
    });
    if (!res.ok) throw new Error(`request failed: ${path}`);
    return res.json().catch(() => ({}));
  }

  return {
    listCampaigns: () => getJson("/v1/campaigns"),
    createCampaign: (body: unknown) => postJson("/v1/campaigns", body),
    closeCampaign: (id: string) => postJson(`/v1/campaigns/${id}/close`, {}),
    // Add more in next tasks.
  };
}
```

**Step 3: Implement CLI**

Create `src/cli/index.ts`:
- Export `parseArgs(argv: string[])`.
- Support `--base-url` (or `FLEET_BASE_URL` env).
- Command shape:
  - `campaign list`
  - `campaign create --name ... --selector ... --action ... [--payload-json '{...}']`
  - `campaign close <id>`
  - `labels set <instanceId> <key> <value>`
  - `groups create --name ... --selector ...`
  - `events export --format jsonl|csv [--out path] [--campaign-id ...] [--instance-id ...]`
  - `artifacts get <id>`
  - `bundles upload --name ... --file path/to.tar.gz`

Keep parsing minimal and strict: unknown flags => exit 2.

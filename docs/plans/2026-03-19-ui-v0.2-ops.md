# UI v0.2 Ops Console Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Upgrade Fleet into an operator-oriented v0.2.x web console by adding backend support for display metadata, task naming, pagination, overview stats, file editing, and then rebuilding the UI around those capabilities.

**Architecture:** Build this in three layers. First extend the schema and API so the UI has real operator-facing primitives (`display_name`, `last_seen_ip`, `task_name`, paginated queries, overview stats, file APIs). Then reorganize the frontend into Chinese-first operator workflows (`概览`, `实例`, `任务与审计`, `技能管理`, `分组与标签`, `批量任务`, `文件与记忆`). Finish by standardizing pagination, contrast, and modal editing interactions. Keep the existing hash router, but replace page boundaries and data flows where needed.

**Tech Stack:** TypeScript, Fastify, React 18, Vite, Postgres, Redis, Vitest, Testing Library.

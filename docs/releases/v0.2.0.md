# v0.2.0 Release Notes

Date: 2026-03-18

## Summary

`v0.2.0` is the first stabilized release for the upgraded Fleet control plane. It consolidates the bulk-management work, the v0.1.1 hardening pass, and the new Fleet-native config patch workflow into a single rollback point.

## Highlights

- Bulk management is now centered on `Labels -> Groups -> Campaigns`.
- `Campaign` execution supports selector-based fan-out, dynamic membership follow-up, per-instance per-generation deduplication, manual close, and historical audit retention.
- Gate evaluation is action-aware:
  - `online` is always required
  - `gateway_reachable` applies to gateway-bound work, except `fleet.gateway.probe`
  - `openclaw_version` is only enforced when `gate.minVersion` is explicitly set
  - `skills_snapshot` only blocks skill-mutating actions
- Audit and analysis are split cleanly:
  - Events keep append-only operator timelines
  - Artifacts keep raw payload/result/error bodies for later inspection
- Skill bundle management is available end-to-end:
  - upload
  - download
  - delete
  - distribute and install through `fleet.skill_bundle.install`
- Browser-facing delete flows use POST aliases where needed to avoid proxy/browser DELETE issues.

## New in v0.2.0

- Added `fleet.config_patch`, a Fleet-native high-level action for batch config changes.
- Sidecar now resolves each target instance's current config hash automatically via `config.get -> config.patch(baseHash)`.
- On OpenClaw stale-hash conflicts, `fleet.config_patch` re-reads config and retries once automatically.
- UI now exposes `fleet.config_patch` in:
  - Campaign creation
  - Task modal
  - Memory/Persona page
- Audit `exec.queued` events now redact config patch `payload.raw`, while the raw payload remains available in artifacts.

## Operator Guidance

- Prefer `fleet.config_patch` for normal batch config updates.
- Keep raw `config.patch` for expert/manual compare-and-swap workflows only.
- `memory.replace` no longer performs an implicit session reset; issue `session.reset` explicitly if you need one.
- `rollout` is stored but still not enforced by the reconciler in this release.

## Recommended Rollback Point

Use tag `v0.2.0` as the rollback baseline for the current upgraded Fleet stack.
